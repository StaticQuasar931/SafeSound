# SafeSound

SafeSound is a static, school-safe music web app for GitHub Pages. It includes a built-in legal starter library, dark mode, playlists, favorites, a lyrics panel for original safe songs, and a browser-based music creator that lets users build their own local tracks without a backend.

## Main Features

- Play, pause, next, previous, seek, volume, shuffle, and repeat
- Responsive library with search, mood/category filters, and favorites
- Local playlists stored in the browser
- Dark mode and Focus Mode
- Original lyric tracks with on-screen lyrics
- SafeSound Studio for creating your own generated songs locally
- Offline-friendly service worker for repeat visits
- Visible About and Licenses section

## What Makes This Safe

- No Spotify, YouTube, Apple Music, or other commercial streaming integrations
- No uploads, comments, chat, or social features
- No copyrighted commercial songs are bundled
- Starter tracks are original SafeSound demo tracks generated by the app
- Custom songs are generated locally in the browser and stored in localStorage

## Project Files

- `index.html` - app structure
- `style.css` - responsive design and theme styling
- `tracks.js` - bundled starter catalog and metadata
- `script.js` - player logic, filters, playlists, lyrics, and music creator
- `service-worker.js` - repeat-visit caching

## GitHub Pages Setup

1. Create or open a GitHub repository.
2. Put all project files in the repository root.
3. Commit and push the files.
4. In GitHub, open `Settings` then `Pages`.
5. Choose your main branch and the root folder.
6. Save and wait for GitHub Pages to publish.

All paths are relative, so the project is GitHub Pages friendly.

## Using SafeSound Studio

The app includes a built-in creator that lets users:

- Name a song
- Choose artist name
- Pick category, tempo, waveform, root note, and pattern
- Add original lyrics or notes
- Preview the generated song
- Save the song locally in the browser

Important: lyrics typed into the studio must be original and school-safe.

## Replacing Demo Tracks With Real Legal Audio

If you want to replace generated demo tracks with real files, use a structure like this:

```text
assets/
  audio/
    calm/
    focus/
    lofi/
    upbeat/
    piano/
    ambient/
    classical/
    lyrics/
  covers/
```

Then update entries in `tracks.js` like this:

```js
{
  id: "gentle-waves",
  title: "Gentle Waves",
  artist: "Example Artist",
  category: "calm",
  mood: "gentle",
  src: "assets/audio/calm/gentle-waves.mp3",
  cover: "assets/covers/gentle-waves.jpg",
  license: "CC0 1.0 Universal",
  sourceName: "Example Archive",
  sourceUrl: "https://example.com/gentle-waves",
  durationHint: 70,
  lyrics: []
}
```

## License Rules For New Music

Only add music that is clearly legal to redistribute in your site or repository:

- Public-domain music
- CC0 music
- Other redistributable royalty-free music only when the license clearly allows bundling and redistribution

Do not add:

- Commercial copyrighted songs
- Music ripped from streaming platforms
- Music with unclear licenses
- User upload features

You should always verify license terms yourself before adding tracks.

## Keyboard Shortcuts

- `Space` - play or pause
- `Left Arrow` - back 5 seconds
- `Right Arrow` - forward 5 seconds
- `F` - toggle Focus Mode
- `D` - toggle dark mode
